Views

View is a basic MineDroid gui class. Views allow you to interact with player and handle mouse and/or key events and represent different states of your mod. You can check out View class javadoc here.

View Measuring

When one of the view's layout width or height parameters are set to wrap_content, parent view should call Layout measure(Point) method of the view. This method accepts maximum available for view width and height (as a Point) and returns measured layout with width and/or height set to values that view desires to have. These values later will be used by parent view to calculate resolved layout of the view which will be passed to the view via void resolveLayout(Layout) method.

Drawing view contents

When parent draws view, it calls it's onDraw method which setups draw area limits for view and calls drawContents method of the view.
When drawing your view, be sure that you use position property for a right placement of the view.

Keyboard and Mouse events

MineDroid keyboard and mouse event system was written from HTML5 event system. Every event propagates downwards in views tree until it will reach the targeted element and, after it, it starts to bubble to the top of the elements hierarchy. Event can be cancelled at any time — it will prevent calling any event handler that are higher on view hierarchy or had been registered later than current event handler.

Subscribing/unsubscribing to an event

To subscribe on keyboard events, you should create an instance of desired event listener interface and pass it to the on(Event) method of the view. You should save your event listener on a field of your class because EventBase uses WeakReference to store your listener. That means your event listener will be unsubscribed and destroyed as soon as the last reference outside of the EventBase (View) class will be removed. If you want to unsubscribe your event listener earlier, you can pass it to the off(Event) method of the View.

Keyboard and Mouse event interfaces

  • OnKeyDown
  • OnKeyUp
  • OnKeyPress
  • OnMouseIn
  • OnMouseMove
  • OnClick
  • OnDblClick
  • OnMouseDown
  • OnMouseUp
  • OnMouseOut

Other events

Class EventBase is a simplified version of OnKiUp android library EventHub class. It allows you to send and receive any events derived from Event class. To use EventBase on your class, you should extend EventBase class.

Descendants

  • ContentView — parent class for all views that have content inside of it:
    • TextView — displays text
      • Button
      • EditText
    • ViewGroup — parent for all layout views:
      • LinearLayout — This view positions it's children in a line;
        • ListView — displays scrollable lists;
      • RelativeLayout — This view provides possibility to position child views relatively to each other;
    • ScrollView — provides vertical scroll for it's content view

Overlays

Overlay is a window (or, in Android — Activity) that can be shown on users screen. To create overlay, you should extend class com.onkiup.minecraft.gui.Overlay and register it with MineDroid.addScreen();

Overlay methods

Callback method Description
void dismiss() Dismisses opened overlay
To create overlay, MineDroid will call it's methods ResourceLocation getContentLayout().
When overlay is being shown, MineDroid will call it's method onStart and onStop — when it is being dismissed.
