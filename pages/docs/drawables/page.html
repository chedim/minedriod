Drawables

Just as Android, MineDroid uses Drawable classes to draw any graphic elements on the screen. Drawables are mostly used as Views backgrounds and other elements.

Currently MineDroid has these Drawables:

  • BitmapDrawable — draws any supported by Minecraft image;
  • ColorDrawable — draws a colored square;
  • EllipseDrawable — draws a colored ellipse (circle in some cases ;-)
  • NinePatchDrawable — draws nine-patch;
  • RoundedCornerDrawable — utilizes ColorDrawable and EllipseDrawable to draw a colored rectangle with rounded corners;
  • StateDrawable — draws different drawables for different view states (pressed, hover, etc);
  • TextDrawable — draws text;
  • TextureDrawable — not implemented yet;
  • BorderDrawable — used to draw view borders in debug mode

Nine patches

NinePatchDrawable uses nine pictures to display a drawable with properly resized border. To use this drawable you should create a subfolder in folder 'assets/<modid>/ninepatches/<nine-patch-name>/' with nine png images named like this:

0.0.png 0.1.png 0.2.png
1.0.png 1.1.png 1.2.png
2.0.png 2.1.png 2.2.png

Current 9patch realization doesn't support image paddings so you should set paddings manually in your view. Ninepatch border size will be taken from corner images sizes (0.0.png, 0.2.png, 2.0.png, 2.2.png). It's very convenient to use Photoshop slices to create nine-patches.

XML drawables

You can define and store drawables as XML-files under directory 'assets/<modid>/drawables/'. Here are some examples:

ColorDrawable: 
<Color xmlns="http://onkiup.com/minecraft/xml"
    color="ff5faed5"/>
TextDrawable: 
<Text xmlns="http://onkiup.com/minecraft/xml"
    text="No items."
    />
RoundedCornerDrawable: 
<RoundedCorner xmlns="http://onkiup.com/minecraft/xml"
        color="66ffffff" radius="5" />
StateDrawable: 
<State xmlns="http://onkiup.com/minecraft/xml">
    <Color color="ff21aae0"/>
    <Color state="hover" color="ff61c7f7"/>
    <Color state="pressed" color="ff5faed5"/>

</State>
