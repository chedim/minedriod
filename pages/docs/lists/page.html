Lists

Work with ListView in MineDroid is a slightly different task than working with it in Android. Unlike Android's ListView, MineDroid's realization doesn't require you to create Adapters — it accepts any Lists of data. However, it uses ViewHolders to create, fill and control views. Every ViewHolder takes responsibility for displaying Views for different list item classes so, every ViewHolder should be registered in ListView with it's class by setHolder(Holder) method. ListView ViewHolders have some additional information so they should be derived from ListView.Holder class.

If ListView will meet any item with class that cannot be processed by any of registered ViewHolders, ListView will omit the item and it will not be shown.

Creating ViewHolder

To create a ViewHolder for ListView, you should extend ListView.Holder class and implement some importants methods:

  • ResourceLocation getViewLocation — returns location of views XML layout file (if you want to create your view in code, you can override View getView() method of the Holder class);
  • void link(View) — here you should accept a new view with which your holder will be associated;
  • void fill(Object) — here you should accept a new object with which your holder will be associated and populate your view with values from this object;
