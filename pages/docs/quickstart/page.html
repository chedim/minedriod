Quick start guide

Example mod

Let's create an example mod with id 'example'! In this mod we will show a simple alert message to our user.

Downloading and installing

You can download a separate jar or checkout sources from github.
MineDroid comes as a separate Forge mod so you don't need to include it into your module jar file.

If you use Gradle, you can add Minedroid from a GitHub hosted maven repository: 

https://raw.github.com/chedim/maven-repo/master/

MineDroid artifact is: 
com.onkiup.minedroid:minedroid:1.0

Also there is a plugin for Gradle that generates R class with links to your mod all resources: 
com.onkiup.gradle:minedroid:1.1-SNAPSHOT

Here is an example gradle.build file:
buildscript {
    repositories {
        maven {
            name = "forge"
            url = "http://files.minecraftforge.net/maven"
        }
        maven {
            name="onkiup"
            url = "https://raw.github.com/chedim/maven-repo/master/";
        }
    }
    dependencies {
        classpath 'net.minecraftforge.gradle:ForgeGradle:1.2-SNAPSHOT'
        classpath 'com.onkiup.gradle:minedroid:1.1-SNAPSHOT'
    }
}
apply plugin: 'forge'
apply plugin: 'minedroid'

dependencies {
            compile 'com.onkiup.minedroid:minedroid:1.0'
}

Creating R class

MineDroid gives you very powerful system of resources handling but it requires some additional steps to be made. MineDroid gradle plugin helps you to automate these steps by adding gradle 'minedroid' task that generates R class with links to all of your mod resources. This task creates a source folder 'gen' in the root folder of your project and you should manually register this source folder with your IDE.

Since this task cannot know your module package name, you should define it in your 'mcmod.info' file as a 'package' field: 

[
{
  "modid": "example",
  "name": "Example MineDroid mod",
  "description": "The brand new GUI.",
  "version": "${version}",
  "mcversion": "${mcversion}",
  "url": "",
  "updateUrl": "",
  "authorList": ["chedim"],
  "credits": "Onkiup",
  "logoFile": "",
  "screenshots": [],
  "package": "com.example.example",
  "dependencies": []
}
]

This task runs every time when you compile your mod with gradle. To run this task manually, simply execute command 'gradle minedroid' in your project directory.

Mod string values

As soon as we want to show some message to the user, we should define it by creating a strings file 'assets/example/values/strings.xml': 

<strings>
    <string name="message">This is our message!</string>
    <string name="dismiss">Close window</string>
</strings>
Later we can use some qualifiers to, in example, localize our mod.

Creating Screen

To create a screen, you should create it's layout first. Layouts are stored as XML files in your 'assets/<module_id>/layouts' folder. Let's assume that out module id will be 'example'. For this quick start, we will create a very simple layout file 'assets/example/layouts/simple.xml' with two views: TextView and Button: 

<RelativeLayout
    xmlns:mc="http://onkiup.com/minecraft/xml"
    mc:width="wrap_content" mc:height="wrap_content" mc:padding="10"
    mc:alignCenter="parent" mc:alignMiddle="parent">
        <TextView mc:id="@id/message" mc:width="wrap_content" mc:height="wrap_content"
            mc:alignCenter="parent"/>
        <Button mc:id="@id/dismiss" mc:width="wrap_content" mc:height="wrap_content"
            mc:alignCenter="parent" mc:below="@id/message" mc:text="@string/dismiss"/>
</RelativeLayout>
Afer we created this resource file, we should execute 'gradle minedroid' in the project directory to generate the R class.

Now we can create a Overlay class that will represent our gui window: 

@SideOnly(Side.CLIENT)
public class ExampleOverlay extends Overlay {

    private View.OnClick onDismissClick = new View.OnClick() {
        @Override
        public void handle(MouseEvent event) {
            dismiss();
        }
    }

    public ExampleOverlay(Context context) {
        super(context);
    }

    @Override
    protected ResourceLocation getContentLayout() {
        return R.layout.example;
    }

    @Override
    protected void fill(View content) {
        TextView message = (TextView) content.findViewById(R.id.message);
        message.setTex(R.string.message);
        Button dismiss = (Button) content.findViewById(R.id.dismiss);
        dismiss.on(onDismissClick);
    }

}

Initializing MineDroid

Now we should initialize MineDroid instance after our module had started: 

@Mod(modid = "example", version = "1.0")
public class Example {

    private static MineDroid md;

    @Mod.EventHandler
    public void init(FMLInitializationEvent event) {
        /* is it a client? */
        try {
            Class.forName("net.minecraft.client.Minecraft");
            NetworkRegistry.INSTANCE.registerGuiHandler(this, md = new MineDroid(R.class));
        } catch (ClassNotFoundException e) {
            System.out.println("Dedicated server.");
        }
    }

}

Showing the ExampleOverlay

Now, to show the window, you should just call MineDroid.open(ExampleOverlay.class);
That's it!
